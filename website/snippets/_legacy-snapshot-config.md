:::info
Начиная с [трека релизов <Constant name="cloud" /> «Latest»](/docs/dbt-versions/cloud-release-tracks) и dbt Core v1.9, определение snapshots в `.sql`‑файле с использованием блока config считается устаревшим методом. Вместо этого вы можете определять snapshots в формате YAML, используя актуальные [конфигурации, специфичные для snapshots](/docs/build/snapshots#configuring-snapshots). Для новых snapshots мы рекомендуем использовать именно эти последние конфигурации. Если вы хотите применить их к уже существующим snapshots, потребуется выполнить [миграцию](#snapshot-configuration-migration).
:::
