:::info `profiles.yml` file is for dbt Core and dbt fusion only
If you're using <Constant name="dbt_platform" />, you don't need to create a `profiles.yml` file. This file is only necessary when you use dbt Core or dbt Fusion locally. To learn more about Fusion prerequisites, refer to [About Fusion installation](/docs/fusion/install-fusion). To connect your data platform to <Constant name="cloud" />, refer to [About data platforms](/docs/cloud/connect-data-platform/about-connections).
:::
