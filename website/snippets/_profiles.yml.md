Если вы используете dbt из командной строки, вам необходим файл `profiles.yml`, который содержит параметры подключения к вашей платформе данных. Система читает файл `dbt_project.yml`, чтобы определить имя `profile`, а затем ищет профиль с таким же именем в файле `profiles.yml`. Этот профиль содержит всю информацию, необходимую dbt для подключения к вашей платформе данных.

:::note Аккаунт dbt platform
Если вы используете облачную платформу <Constant name="cloud" />, вы можете [подключить вашу платформу данных](/docs/cloud/connect-data-platform/about-connections) напрямую в интерфейсе <Constant name="cloud" /> и вам не понадобится файл `profiles.yml`.
:::

В этом разделе описаны части файла `profiles.yml`, которые не зависят от конкретной платформы данных. Подробности о настройке подключения см. на странице, соответствующей вашей платформе данных.

Чтобы добавить дополнительный target (например, `prod`) в существующий `profiles.yml`, вы можете добавить ещё одну запись под ключом `outputs`.

<File name='profiles.yml'>

```yml

<profile-name>:
  target: <target-name> # это target по умолчанию
  outputs:
    <target-name>:
      type: <bigquery | postgres | redshift | snowflake | other>
      schema: <schema_identifier>
      threads: <natural_number>

      ### параметры подключения, специфичные для базы данных
      ...

    <target-name>: # дополнительные targets
      ...

<profile-name>: # дополнительные профили
  ...

```

</File>

## Пользовательская конфигурация

Вы можете задать значения по умолчанию для глобальных конфигураций, которые будут применяться ко всем проектам, запускаемым на вашем локальном компьютере. Подробнее см. в разделе [About global configs](/reference/global-configs/about-global-configs).
