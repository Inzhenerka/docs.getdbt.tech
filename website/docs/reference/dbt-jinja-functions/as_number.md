Jinja-фильтр `as_number` приводит результат компиляции Jinja к числовому
значению (целому числу или числу с плавающей точкой) либо возвращает ошибку,
если его невозможно представить в виде числа.

### Использование {#usage}

В примере ниже фильтр `as_number` используется для приведения значения
переменной окружения к числовому типу, чтобы динамически управлять портом
подключения.

<File name='profiles.yml'>

```yml
my_profile:
  outputs:
    dev:
      type: postgres
      port: "{{ env_var('PGPORT') | as_number }}"
```

</File>
