[Конфликты слияния](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/addressing-merge-conflicts/about-merge-conflicts) в [<Constant name="cloud_ide" />](/docs/cloud/studio-ide/develop-in-studio) часто возникают, когда несколько пользователей одновременно вносят правки в один и тот же участок одного и того же файла. В этом случае <Constant name="git" /> становится сложно определить, какие изменения нужно включить в итоговое слияние.  

Процесс разрешения конфликтов слияния позволяет пользователям выбрать, какие строки кода нужно сохранить и зафиксировать (commit). В этом документе показано, как разрешать конфликты слияния в <Constant name="cloud_ide" />.

## Определение конфликтов слияния {#identify-merge-conflicts}

С конфликтом слияния можно столкнуться в двух случаях:

- При подтягивании изменений из основной ветки, когда кто-то другой уже смержил конфликтующее изменение.
- При коммите ваших изменений в ту же ветку, когда другой пользователь уже успел закоммитить свои изменения раньше.

Способ [разрешения](#resolve-merge-conflicts) конфликта в обоих сценариях будет абсолютно одинаковым. 

Например, если вы и ваш коллега внесли изменения в один и тот же файл и закоммитили их, вы столкнетесь с конфликтом слияния сразу после нажатия **Commit and sync**. 

<Constant name="cloud_ide" /> отобразит следующее:

- **Commit and resolve** в git action bar в разделе **Version Control** вместо **Commit** &mdash; это означает, что Cloud <Constant name="cloud_ide" /> обнаружил конфликты, которые необходимо разрешить.
- Редактор с разделением на две части &mdash; левая часть содержит ваши изменения и доступна только для чтения. Правая часть содержит дополнительные изменения, позволяет редактирование и помечает конфликт специальными маркерами:

```
<<<<<< HEAD
    your current code
======
    conflicting code
>>>>>> (some branch identifier)
```
- Файл и путь к нему, подсвеченные красным цветом в **File <Constant name="explorer" />**, с иконкой предупреждения, указывающей на необходимость разрешения конфликта.
- Имя файла, выделенное красным цветом в разделе **Changes**, также с иконкой предупреждения.
- Если вы попытаетесь выполнить commit, не разрешив конфликт, <Constant name="cloud_ide" /> покажет всплывающее окно со списком файлов, которые требуется разрешить.

## Разрешение конфликтов слияния {#resolve-merge-conflicts}
Вы можете легко разрешать конфликты слияния, связанные с конкурирующими изменениями строк, прямо в Cloud <Constant name="cloud_ide" />.

1. В <Constant name="cloud_ide" /> вы можете редактировать правую часть конфликтующего файла, выбрать строки кода, которые нужно сохранить, и удалить остальные. 
    * Примечание: левая часть редактора доступна только для чтения, и вносить в неё изменения нельзя.
3. Удалите специальные флаги или маркеры конфликта `<<<<<<<`, `=======`, `>>>>>>>`, которые обозначают конфликт слияния, а также выберите строки кода, которые необходимо сохранить.
4. Если в файле больше одного конфликта слияния, прокрутите вниз до следующего набора маркеров конфликта и повторите шаги один и два, чтобы разрешить конфликт.
5. Нажмите **Save**. Вы заметите, что подсветка строк исчезнет и фон станет обычным. Это означает, что конфликт был успешно разрешен.
6. Повторите этот процесс для каждого файла, в котором есть конфликт слияния.


<Lightbox src="/img/docs/dbt-cloud/cloud-ide/resolve-conflict.png" title="Выбор строк кода, которые нужно сохранить"/>

:::info Редактирование файлов с конфликтами
- Если вы откроете файл с конфликтом из раздела **Changes**, имя файла будет отображаться примерно как `model.sql (last commit)`, и файл будет полностью доступен только для чтения, без возможности редактирования. <br />
- Если вы откроете файл с конфликтом из **File <Constant name="explorer" />**, вы сможете редактировать файл в правой части редактора.
:::

## Коммит изменений {#commit-changes}

После того как все конфликты слияния будут разрешены, последним шагом станет коммит внесенных изменений.

1. Нажмите в git action bar кнопку **Commit and resolve**. 
2. Во всплывающем окне **Commit Changes** будет подтверждено, что все конфликты были разрешены. Введите сообщение коммита и нажмите **Commit Changes**.
3. <Constant name="cloud_ide" /> вернется в обычное состояние, и вы сможете продолжить разработку!
