[Конфликты слияния](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/addressing-merge-conflicts/about-merge-conflicts) в [<Constant name="cloud_ide" />](/docs/cloud/studio-ide/develop-in-studio) часто возникают, когда несколько пользователей одновременно вносят изменения в один и тот же раздел одного и того же файла. В таких случаях <Constant name="git" /> становится сложно определить, какие изменения нужно включить в итоговое слияние.  

Процесс разрешения конфликтов слияния позволяет пользователям выбрать, какие строки кода они хотят сохранить и закоммитить. В этом документе показано, как разрешать конфликты слияния в <Constant name="cloud_ide" />.

## Определение конфликтов слияния

Вы можете столкнуться с конфликтом слияния двумя способами:

- При получении изменений из вашей основной ветки, когда кто-то другой уже слил конфликтующее изменение.
- При фиксации ваших изменений в ту же ветку, когда кто-то другой уже зафиксировал свои изменения первым.

Способ [разрешения](#resolve-merge-conflicts) любой из этих ситуаций будет точно таким же.

Например, если вы и ваш коллега вносите изменения в один и тот же файл и фиксируете их, вы столкнетесь с конфликтом слияния, как только нажмете **Commit and sync**.

<Constant name="cloud_ide" /> отобразит следующее:

- **Commit and resolve** — панель действий git в разделе **Version Control** вместо **Commit** &mdash; это означает, что Cloud <Constant name="cloud_ide" /> обнаружила конфликты, которые необходимо разрешить.
- Режим редактора с разделением на две части (2-split) &mdash; левая часть содержит ваши изменения кода и доступна только для чтения. Правая часть содержит дополнительные изменения, позволяет редактировать их и помечает конфликт с помощью специальных флагов:

```
<<<<<< HEAD
    ваш текущий код
======
    конфликтующий код
>>>>>> (идентификатор ветки)
```
- Файл и путь, выделенные красным цветом в **File <Constant name="explorer" />**, с иконкой предупреждения — это указывает на файлы, которые необходимо разрешить.
- Имя файла, выделенное красным цветом в разделе **Changes**, с иконкой предупреждения.
- Если вы нажмёте commit, не разрешив конфликт, <Constant name="cloud_ide" /> покажет всплывающее окно со списком файлов, которые требуется разрешить.

## Разрешение конфликтов слияния

## Разрешение конфликтов слияния
Вы можете без труда разрешать конфликты слияния, связанные с конкурирующими изменениями строк, в Cloud <Constant name="cloud_ide" />.

1. В <Constant name="cloud_ide" /> вы можете редактировать правую часть файла с конфликтом, выбрать, какие строки кода нужно сохранить, а остальные удалить.  
    * Примечание: левый редактор доступен только для чтения, и вы не можете вносить в нём изменения.
3. Удалите специальные флаги или маркеры конфликтов `<<<<<<<`, `=======`, `>>>>>>>`, которые обозначают конфликт слияния, а также выберите, какие строки кода нужно сохранить.
4. Если в файле есть более одного конфликта слияния, прокрутите вниз до следующего набора маркеров конфликта и повторите шаги один и два, чтобы разрешить конфликт.
5. Нажмите **Save**. Вы заметите, что подсветка строк исчезнет и фон станет обычным. Это означает, что конфликт был успешно разрешён.
6. Повторите этот процесс для каждого файла, в котором есть конфликт слияния.

:::info Edit conflict files
- Если вы откроете файл с конфликтом в разделе **Changes**, имя файла будет отображаться примерно как `model.sql (last commit)`. Такой файл полностью доступен только для чтения и не может быть отредактирован. <br />
- Если вы откроете файл с конфликтом через **File <Constant name="explorer" />**, вы сможете редактировать файл в правой панели.
:::
:::

## Фиксация изменений

Когда вы разрешили все конфликты слияния, последним шагом будет фиксация внесенных вами изменений.

1. Нажмите на панель действий git **Commit and resolve**.
2. Во всплывающем окне **Commit Changes** будет подтверждено, что все конфликты разрешены. Введите сообщение коммита и нажмите **Commit Changes**.
3. <Constant name="cloud_ide" /> вернётся в обычное состояние, и вы сможете продолжить разработку!
