Чтобы настроить и сконфигурировать права доступа на уровне окружения, у вас должны быть права на запись в настройках **Groups & Licenses** вашего аккаунта <Constant name="cloud" />. Подробнее о ролях и разрешениях см. в разделе [User permissions and licenses](/docs/cloud/manage-access/seats-and-users).

Разрешения на уровне окружения не совпадают с [управлением доступом на основе ролей (RBAC)](/docs/cloud/manage-access/about-user-access#role-based-access-control) на уровне учетной записи и настраиваются отдельно от этих рабочих процессов.

## Инструкции по настройке

В вашем аккаунте <Constant name="cloud" />:

1. Нажмите на название вашего аккаунта над иконкой профиля на левой боковой панели. Затем выберите **Account settings**.
2. Перейдите в раздел **Groups & Licenses**. Мы рекомендуем удалить группы по умолчанию `Owner`, `Member` и `Everyone` и вместо этого назначать пользователей в ваши организационные группы, чтобы не предоставлять им избыточные повышенные привилегии.

   Однако перед удалением этих групп убедитесь, что все существующие пользователи — включая вас — переназначены в соответствующие организационные группы. Вы не сможете удалить группу `Owner`, пока _как минимум_ один пользователь не будет добавлен в другую группу с набором разрешений администратора аккаунта либо пока не будет пользователя с IT‑лицензией. Этот защитный механизм гарантирует, что в аккаунте всегда есть администратор, способный управлять изменениями групп.

<Lightbox src="/img/docs/dbt-cloud/groups-and-licenses.png" width="80%" title="Groups & Licenses page in dbt with the default groups highlighted."/>

3. Создайте новую группу или откройте существующую. Если это новая группа, задайте ей имя, затем прокрутите страницу до раздела **Access & permissions**. Нажмите **Add permission**.

<Lightbox src="/img/docs/dbt-cloud/add-permissions.png" width="80%" title="Раздел Access & permissions с выделенной кнопкой Add."/>

4. Выберите **Permission set** для группы. Настроить разрешения на уровне окружения можно **только** для следующих наборов разрешений:
    - Database admin
    - <Constant name="git" /> admin
    - Team admin
    - Analyst
    - Developer

   Other permission sets are restricted because they have access to everything (for example, Account admin), or limitations prevent them from having write access to environments (for example, Account viewer).

Если вы выберете набор разрешений, который не поддерживается, параметр разрешений для среды не будет отображаться.

<Lightbox src="/img/docs/dbt-cloud/no-option.png" width="80%" title="Вид окна разрешений, если нет опции для разрешений на уровне окружения."/>

5. Выберите **Environment** для доступа группы. По умолчанию используется **All environments**, но вы можете выбрать несколько. Если ни один вариант не выбран, группа будет иметь доступ только для чтения.

<Lightbox src="/img/docs/dbt-cloud/environment-options.png" width="80%" title="Список доступных окружений с выделенными полями Staging и General."/>

6. Сохраните **Group settings**. Теперь всё настроено, и вы готовы назначать пользователей!

## Пользовательский опыт

Пользователи с разрешениями на окружение увидят все возможности, назначенные их роли. Разрешения на уровне окружения предоставляют доступ `write` или `read-only`. В настоящее время эта функция не поддерживает определение, какие функции в окружении доступны. Для получения более подробной информации о том, что можно и нельзя делать с разрешениями на уровне окружения, обратитесь к [О разрешениях на уровне окружения](/docs/cloud/manage-access/environment-permissions).
