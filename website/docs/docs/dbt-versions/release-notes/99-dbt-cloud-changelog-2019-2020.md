Добро пожаловать в журнал изменений за 2019 и 2020 годы для приложения <Constant name="cloud" />! В этом журнале вы можете ознакомиться с основными нововведениями, исправлениями и улучшениями, которые были реализованы за этот период.

## dbt Cloud v1.1.16 (23 декабря 2020)

В этом релизе добавлена предварительная поддержка Databricks Spark в <Constant name="cloud" />, а также добавлены два новых набора разрешений для корпоративных аккаунтов.

#### Улучшения

- Добавлена предварительная поддержка Databricks Spark
- Добавлены два новых набора разрешений для корпоративных аккаунтов: Account Viewer и Project Creator

#### Исправления

- Улучшена инфраструктура логирования для dbt run logs
- Исправлены ошибки логирования SSH туннеля


## dbt Cloud v1.1.15 (10 декабря 2020)

На этот раз есть много отличных вещей, которые стоит обсудить: на этой неделе всё действительно сложилось воедино! Много воодушевления вызвало добавление Spark в семейство подключений, а также закрытие нескольких давно существовавших багов.

#### Улучшения

- Добавлен Spark в качестве опции для настройки базы данных

#### Исправления

- Исправлен один сложный баг, из‑за которого один email мог быть связан с несколькими пользовательскими аккаунтами
- Исправлена маршрутизация react‑страницы setup‑connection
- Вынесена логика выбора групп из типов лицензий и названий групп
- Добавлена обработка ошибок JSON при разборе тела запросов v1/v2
- Корректная обработка AuthForbidden и AuthCancelled — т.е. без выбрасывания 500‑х ошибок
- Исправлена регрессия с индикатором загрузки для `<Constant name="cloud_ide" />`


## dbt Cloud v1.1.14 (25 ноября 2020)

В этом релизе добавлено несколько новых связующих элементов, в частности OAuth для BigQuery и доработки SparkAdapter. Также присутствуют некоторые улучшения качества жизни и заделы на будущее, ориентированные на наших любимых пользователей <Constant name="cloud_ide" />, а также улучшенная передача данных для наблюдаемости — в системы управления логами и для использования API.

#### Улучшения

- Обновление allowlist IP-адресов  
- Пользователь может использовать OAuth для BigQuery в учётных данных профиля  
- Добавление backend-моделей, мапперов и сервисов SparkAdapter  
- Добавлена интеграция OAuth для BigQuery  
- Добавление индекса БД для `owner_thread_id`

#### Исправления

- Исправлена ошибка post /run
- Исправлена ошибка, когда неверный аргумент передавался в dbt runs
- Логирование необработанных ошибок в контекстном менеджере переменных окружения
- Удалены разрешения настроек аккаунта для интеграций пользователей

В этом релизе добавлена поддержка запуска runs с переопределёнными атрибутами через API‑эндпоинт [triggerRun](/dbt-cloud/api-v2#).

- Использовать кэширование S3‑клиента для повышения скорости загрузки логов для запланированных запусков
- Добавить поддержку запуска заданий [с переопределёнными атрибутами через API](/dbt-cloud/api-v2#)
- Прояснить формулировки, связанные с «upgrade», на странице биллинга

#### Исправления

- Эндпоинт групп GitLab теперь возвращает все группы и подгруппы
- Поддержка конфигураций повторных попыток BigQuery со значением 0
- Предотвращение сбоев веб-IDE после выполнения неверной команды dbt
- Применение дополнительной очистки логов для фильтрации краткосрочных git-учетных данных
- Исправление старой миграции для поля auth_url, чтобы сделать его nullable
- Поддержка путей в URL-адресе экземпляра GitLab
- Исправление для URL-запроса токена аутентификации в потоке oauth GitLab

## dbt Cloud v1.1.9 (1 октября 2020)

Этот выпуск добавляет возможность для администраторов на корпоративном плане настраивать разрешения управления доступом на основе ролей, применяемые к проектам в их аккаунте. Кроме того, теперь доступна отсрочка выполнения заданий за флагом функции, а также выпущен ряд исправлений и улучшений.

#### Улучшения

- Добавлена версия <Constant name="cloud" /> в боковую панель навигации
- Добавлены интерфейсы просмотра, создания и изменения прав групп RBAC
- Добавлена персональная git-аутентификация для модальных окон обработки ошибок <Constant name="cloud_ide" />
- Добавлены Develop Requests в backend-представления
- Реализована отложенная (deferred) обработка выполнения заданий
- Добавлена поддержка dbt v0.18.1b2

#### Исправления

- Исправлен сценарий, при котором взаимодействие с кнопкой Refresh в <Constant name="cloud_ide" /> приводило к тому, что файл `index.lock` оставался в файловой системе <Constant name="cloud_ide" />
- Добавлена проверка URL PR на попытки XSS
- Устранены несоответствия в RBAC
- Исправлена проблема, из-за которой пользователи не могли обновить свой пароль <Constant name="cloud" /> непосредственно в приложении
- Исправление применения пользовательских разрешений для нескольких аккаунтов после аутентификации через SSO
- Google API: по умолчанию используется общий API endpoint, но добавлена возможность переопределения
- Исправлена проблема с отсутствующей переменной email в debug-логировании GSuite
- Уничтожение сессии <Constant name="cloud_ide" /> при переключении проектов
