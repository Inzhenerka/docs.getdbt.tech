Добро пожаловать в журнал изменений за 2019 и 2020 годы для приложения <Constant name="cloud" />! Этот журнал можно использовать, чтобы увидеть основные нововведения, исправления и улучшения за указанный период.

## dbt Cloud v1.1.16 (23 декабря 2020)

В этом релизе добавлена предварительная поддержка Databricks Spark в <Constant name="cloud" />, а также добавлены два новых набора прав доступа для корпоративных аккаунтов.

#### Улучшения

- Добавлена предварительная поддержка Databricks Spark
- Добавлены два новых набора прав доступа для Enterprise: Account Viewer и Project Creator

#### Исправления

- Улучшена инфраструктура логирования для логов dbt run
- Исправлены ошибки логирования SSH-туннелей


## dbt Cloud v1.1.15 (10 декабря 2020)

В этом релизе много отличных новостей: на этой неделе всё действительно сложилось воедино. Большое внимание было уделено добавлению Spark в список поддерживаемых подключений, а также устранению ряда давних ошибок.

#### Улучшения

- Добавлен Spark как вариант при настройке базы данных

#### Исправления

- Исправлена сложная ошибка, из‑за которой один email мог быть связан с несколькими пользовательскими аккаунтами
- Исправлена маршрутизация react-страницы setup-connection
- Логика выбора групп вынесена отдельно от типов лицензий и названий групп
- Обработка ошибок JSON при парсинге тела v1/v2
- Корректная обработка AuthForbidden и AuthCancelled — без возврата 500 ошибок
- Исправлена регрессия с индикатором загрузки <Constant name="cloud_ide" />


## dbt Cloud v1.1.14 (25 ноября 2020)

Этот релиз добавляет несколько важных связующих элементов, включая OAuth для BigQuery и работу над SparkAdapter. Также включены улучшения удобства использования и инвестиции в будущее, ориентированные на пользователей <Constant name="cloud_ide" />, а также улучшена передача данных для наблюдаемости в системах управления логами и использовании API.

#### Улучшения

- Обновлён IP allowlist
- Пользователь может использовать OAuth для BigQuery в учетных данных профиля
- Добавлены backend-модели, мапперы и сервисы SparkAdapter
- Добавлена интеграция BigQuery OAuth
- Добавлен индекс БД для owner_thread_id

#### Исправления

- Исправлен высокий уровень ошибок для POST /run
- Исправлена ошибка передачи некорректного аргумента в dbt runs
- Логирование необработанных ошибок в контексте менеджера переменных окружения
- Удалены права доступа к настройкам аккаунта для пользовательских интеграций


## dbt Cloud v1.1.13 (12 ноября 2020)

В этом релизе добавлена поддержка запуска заданий с переопределёнными атрибутами через API-эндпоинт
[triggerRun](/dbt-cloud/api-v2#). Кроме того, исправлено множество ошибок и внесены улучшения производительности.

#### Улучшения

- Улучшена обработка ошибок для длительных запросов в <Constant name="cloud_ide" />
- Использование кэширования S3 клиента для ускорения загрузки логов запланированных запусков
- Поддержка запуска заданий [с переопределёнными атрибутами через API](/dbt-cloud/api-v2#)
- Уточнён текст про «upgrade» на странице биллинга

#### Исправления

- Эндпоинт GitLab groups теперь возвращает все группы и подгруппы
- Поддержка конфигураций повторных попыток BigQuery со значением 0
- Предотвращено падение web IDE после запуска некорректной команды dbt
- Дополнительная очистка логов для фильтрации краткоживущих git‑учётных данных
- Исправлена старая миграция, чтобы поле auth_url могло быть nullable
- Поддержка путей в URL GitLab instance
- Исправлен URL запроса auth token в GitLab OAuth flow


## dbt Cloud v1.1.12 (30 октября 2020)

В этом релизе в dbt Cloud добавлены dbt v0.18.1 и 0.19.0b1. Также исправлено несколько ошибок.

#### Улучшения

- Обновлён текст на странице биллинга для выбора тарифа по окончании триала
- Улучшена авторизация для Metadata API
- Добавлен dbt 0.19.0b1
- Добавлен dbt 0.18.1

#### Исправления

- Исправлена проблема, при которой группы из других залогиненных аккаунтов отображались в UI RBAC
- Исправлены запрашиваемые GitLab scopes и ошибка при шифровании deploy tokens для GitLab auth
- Исправлена ошибка, при которой null-символы в логах вызывали сбои в запланированных запусках


## dbt Cloud v1.1.11 (15 октября 2020)

Релиз v1.1.11 включает улучшения удобства использования, правки текстов и исправления ошибок. Также это последний раз, когда в номере релиза четыре одинаковые цифры подряд — до версии v2.2.22.

#### Улучшения

- Добавлена обработка исключений InterfaceError для команд
- My Account переименовано в Profile
- Добавлены project и connection в admin backend

#### Исправления

- Устранены ошибки, связанные с наличием null-символов в логах
- Backend для подтверждения email
- Откат run.serialize
- Исправлена ошибка при сериализованном run
- Исправлена логическая ошибка в настройке соединений
- Исправлена ошибка в GitLab auth flow для неаутентифицированных пользователей
- Исправлена ошибка, из‑за которой Native Okta SSO использовал неверный порт


## dbt Cloud v1.1.10 (8 октября 2020)

Этот релиз добавляет поддержку репозиториев, импортированных через GitLab (Enterprise), и включает ряд исправлений и улучшений в <Constant name="cloud_ide" />.

#### Улучшения

- Добавлена интеграция GitLab (Enterprise)
- Добавлена настройка репозитория GitLab в процессе настройки проекта (Enterprise)
- Добавлена автоматическая установка GitLab Deploy Token (Enterprise)
- Добавлен dbt 0.18.1rc1

#### Исправления

- Исправлена ошибка, при которой <Constant name="cloud_ide" /> зависал после смены репозитория проекта
- Исправлено состояние гонки, при котором подключения могли добавляться к неверному проекту
- Исправлено отзыв приглашений по email
- Исправлена ошибка в slim CI deferring run search, когда отсутствие предыдущего запуска вызывало ошибку планировщика
- Исправлен источник нестабильности <Constant name="cloud_ide" />
- Корректная очистка backend <Constant name="cloud_ide" /> при завершении работы
- Всегда отображаются SSO mappings на странице Group Details


## dbt Cloud v1.1.9 (1 октября 2020)

В этом релизе администраторы на тарифе Enterprise получили возможность настраивать права Role Based Access Control, применяемые к проектам в аккаунте. Также отложенное выполнение заданий стало доступно за feature flag, и были выпущены различные исправления и улучшения.

#### Улучшения

- Добавлена версия <Constant name="cloud" /> в боковой панели навигации
- Добавлены UI для просмотра, создания и изменения RBAC Group Permission
- Добавлена персональная git-аутентификация для модальных окон ошибок <Constant name="cloud_ide" />
- Добавлены Develop Requests в backend views
- Реализовано отложенное выполнение заданий
- Добавлена поддержка dbt v0.18.1b2

#### Исправления

- Исправлен сценарий, при котором взаимодействие с кнопкой Refresh <Constant name="cloud_ide" /> приводило к сохранению файла index.lock в файловой системе <Constant name="cloud_ide" />
- Проверка PR URL на XSS
- Устранены несоответствия RBAC
- Исправлена ошибка, из‑за которой пользователи не могли обновить пароль <Constant name="cloud" /> внутри приложения
- Исправлено применение пользовательских прав в нескольких аккаунтах после SSO-аутентификации
- Google API: по умолчанию используется общий API endpoint с возможностью переопределения
- Исправлена ошибка отсутствия email переменной в GSuite debug логировании
- Уничтожение сессии <Constant name="cloud_ide" /> при переключении проектов

#### Улучшения
- Добавлен dbt 0.18.0
- Добавлена нативная поддержка Okta SSO

- Добавлена поддержка git clone через GitLab deploy tokens для запланированных запусков (скоро)
- Добавлены UI страниц RBAC Groups Detail и Groups List

#### Исправления

- Разрешены переменные окружения `*_proxy` в запланированных запусках
