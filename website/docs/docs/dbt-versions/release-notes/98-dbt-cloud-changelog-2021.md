## dbt Cloud v1.1.39 (10 ноября 2021)
Мы выпустили поддержку переменных окружения в <Constant name="cloud" />. Переменные окружения позволяют отделить код от конфигурации — настраивать конфигурацию в зависимости от контекста и безопасно хранить секреты, такие как git‑токены.

#### Новые продукты и возможности
- Теперь вы можете добавлять переменные окружения в свой проект <Constant name="cloud" />. Почему это важно? Переменные окружения — это фундаментальный строительный блок проекта dbt, который до этого момента был доступен только в <Constant name="core" />. Они поддерживают множество сценариев использования, таких как клонирование приватных пакетов, ограничение объёма данных, обрабатываемых в средах разработки, изменение источников данных в зависимости от окружения и многое другое. Подробнее о переменных окружения читайте в нашем [блог‑посте](https://blog.getdbt.com/introducing-environment-variables-in-dbt-cloud/) или [документации](/docs/build/environment-variables).

## dbt Cloud v1.1.37 (13 октября 2021)
dbt v0.21 теперь доступен в dbt Cloud. Главное изменение этого релиза — появление команды `dbt build`. `dbt build` логически выполняет всё, что вы хотите сделать в своём DAG: запускает модели, выполняет тесты, снапшоты и сиды. Он делает это по ресурсам, слева направо по вашему DAG. `dbt build` — это осознанно «мнение‑ориентированная» задача. Это кульминация всего, что мы построили: запуск моделей с устойчивыми <Term id="materialization">материализациями</Term>, приоритизация качества данных с помощью тестов, обновление фикстур через seeds, захват медленно изменяющихся измерений с помощью snapshots. Попробуйте!

#### Новые продукты и возможности
- У нас появилась новая beta‑функция — Model Bottlenecks. Она позволяет визуально увидеть, сколько времени занимает сборка моделей в каждом запуске, чтобы сразу понять, какие модели выполняются дольше всего. Если вам интересно узнать больше, загляните в канал #beta-feedback-model-bottlenecks в Slack‑сообществе dbt, и мы добавим вас в beta‑программу.

## dbt Cloud v1.1.32 (4 августа 2021)
Metadata API теперь в GA! Когда <Constant name="cloud" /> выполняет такие команды, как run, test, seed и другие, dbt генерирует метаданные в виде [artifacts](/reference/artifacts/dbt-artifacts). Эти артефакты содержат массу информации о настройке проекта, времени выполнения, деталях тестов, скомпилированном SQL и многом другом. Теперь <Constant name="cloud" /> предоставляет GraphQL API, который поддерживает произвольные запросы к этим артефактам, позволяя практически мгновенно получать нужные метаданные.

#### Новые продукты и возможности
- Metadata API — это начало нашего набора продуктов для работы с метаданными. Подробнее о том, как использовать Metadata API, читайте [здесь](/docs/dbt-cloud-apis/discovery-api).
- Клиенты dbt Enterprise, использующие GitHub, теперь получают более <Term id="grain">детализированный</Term> контроль доступа в своих dbt‑проектах. dbt будет применять git‑права для каждого разработчика, гарантируя, что политики чтения/записи в GitHub корректно применяются и в IDE.

## dbt Cloud v1.1.28 (9 июня 2021)
Мы выпустили значительно улучшенный опыт для пользователей GitLab. Обязательно ознакомьтесь с новыми CI‑возможностями, которые теперь доступны клиентам GitLab. Кроме того, всем разработчикам стоит попробовать Slim CI — это существенно ускорит сборку моделей.

#### Новые продукты и возможности

- `Slim CI`: Slim CI теперь доступен всем нашим cloud‑клиентам! С Slim CI вам не нужно пересобирать и тестировать все модели — вы можете настроить dbt Cloud так, чтобы job’ы запускались только для изменённых или новых ресурсов. Если вы используете GitHub или GitLab, попробуйте создать job, который запускается на pull request’ах, и укажите аргумент `state:modified+`. Подробнее о Slim CI читайте [здесь](/docs/deploy/continuous-integration).

- Нативная аутентификация GitLab для <Constant name="cloud" /> Developer и Team: мы вывели нативную GitLab‑аутентификацию в GA. Теперь вы можете импортировать новые GitLab‑репозитории в пару кликов, запускать CI‑сборки при открытии Merge Request’ов и передавать GitLab‑права в git‑действия <Constant name="cloud_ide" />. Инструкции по настройке доступны [здесь](/docs/cloud/git/connect-gitlab).

#### Исправления ошибок
- Разрешён выбор артефактов из job’ов, выполняющих source freshness, если в настройках execution source freshness установлено значение `ON`.
- Исправлено корректное переключение кнопки `RUN ONLY ON CUSTOM BRANCH?`.
- Сохранение информации во вкладке `Statement` при обновлении страницы.
- Несохранённые изменения в <Constant name="cloud_ide" /> теперь сохраняются при коммите.
- Удаление временных схем в хранилище данных для закрытых или смерженных GitLab merge request’ов.

#### Улучшения производительности и доработки
- За кулисами мы постепенно уходим от Angular к React. Мы начали миграцию ключевых частей UI, первой из которых стала основная навигация. Мы ожидаем, что это существенно снизит количество UI‑багов и улучшит пользовательский опыт.
- Добавлена поддержка dbt 0.19.2rc2 и 0.20.0rc1 в <Constant name="cloud" />.
