Отслеживайте свои задания <Constant name="cloud" />, чтобы выявлять возможности для улучшения и настраивать оповещения, которые будут проактивно уведомлять нужных людей или команды.

В этом разделе документации рассматриваются различные возможности <Constant name="cloud" />, которые помогают мониторить выполнение заданий и настраивать оповещения для обеспечения бесперебойной оркестрации, включая:

- [Визуализация и оркестрация downstream‑exposures](/docs/deploy/orchestrate-exposures) <Lifecycle status="managed,managed_plus" /> &mdash; Автоматическая визуализация и оркестрация exposures из дашбордов, а также проактивное обновление базовых источников данных во время запланированных заданий dbt.
- [Использование артефактов](/docs/deploy/artifacts) &mdash; <Constant name="cloud" /> генерирует и сохраняет артефакты для вашего проекта, которые используются для таких возможностей, как создание документации проекта и отчетность по freshness источников.
- [Уведомления о заданиях](/docs/deploy/job-notifications) &mdash; Получайте уведомления по email или в Slack, когда выполнение задания завершается успешно, проходит с предупреждениями, завершается с ошибкой или отменяется.
- [Уведомления о моделях](/docs/deploy/model-notifications) &mdash; Получайте email‑уведомления о любых проблемах, с которыми сталкиваются ваши модели и тесты, сразу после их возникновения во время выполнения задания.
- [Повторный запуск заданий](/docs/deploy/retry-jobs) &mdash; Перезапускайте задания с ошибками с самого начала или с точки сбоя.
- [Видимость запусков](/docs/deploy/run-visibility) &mdash; Просматривайте историю запусков, чтобы понять, где можно улучшить запланированные задания.
- [Freshness источников](/docs/deploy/source-freshness) &mdash; Контролируйте управление данными, включая snapshots для фиксации freshness ваших источников данных.
- [Webhooks](/docs/deploy/webhooks) &mdash; Используйте webhooks для отправки событий о статусах заданий dbt в другие системы.

Чтобы настроить и добавить data health tiles для просмотра freshness данных и проверок качества данных на вашем дашборде, обратитесь к разделу [data health tiles](/docs/explore/data-tile).
