Вы можете просматривать историю выполнения и панель времени моделей, чтобы определить, где можно улучшить задания.

## История выполнения

Панель **Run history** в <Constant name="cloud" /> помогает отслеживать состояние вашего dbt‑проекта. Она предоставляет детальный обзор всех запусков заданий (job runs) в проекте и предлагает набор фильтров, которые позволяют сосредоточиться на конкретных аспектах. С её помощью вы также можете просматривать недавние запуски, находить запуски с ошибками и отслеживать прогресс выполняющихся запусков. Доступ к панели осуществляется из верхнего навигационного меню: нажмите **Deploy**, а затем **Run history**.

Разработчики <Constant name="cloud" /> могут получать доступ к истории своих запусков за последние 365 дней через пользовательский интерфейс (UI) и API <Constant name="cloud" />.

dbt Labs ограничивает самостоятельное получение метаданных истории запусков периодом в 365 дней, чтобы повысить производительность <Constant name="cloud" />.

## Детали выполнения задания

На панели **История выполнения** выберите выполнение, чтобы просмотреть полные детали о нём. Страница деталей выполнения задания отображает триггер задания, commit SHA, время, проведенное в очереди планировщика, все шаги выполнения и их [логи](#access-logs), [время моделей](#model-timing) и многое другое.

Нажмите **Rerun now**, чтобы немедленно перезапустить задание.

### Вкладка Compare <Lifecycle status="managed,managed_plus" />

Вкладка **Compare** отображается для [запусков CI jobs](/docs/deploy/ci-jobs), у которых включена настройка **Run compare changes**. На ней показываются подробности [изменений, которые dbt выявил в результате сравнения](/docs/deploy/advanced-ci#compare-changes) между тем, что находится в вашем production-окружении, и pull request. Чтобы упростить визуальное восприятие различий, <Constant name="cloud" /> подсвечивает изменения в ваших моделях красным цветом (удаления) и зелёным цветом (добавления).

- **Overview** — Краткое резюме изменений в моделях, например количество добавленных или удалённых первичных ключей. 
- **Primary keys** — Подробности об изменениях записей.
- **Modified rows** — Подробности об изменённых строках. Нажмите **Show full preview**, чтобы отобразить все столбцы.
- **Columns** — Подробности об изменениях столбцов. 

Чтобы подробнее просмотреть зависимости и связи ресурсов в вашем проекте, нажмите **View in <Constant name="explorer" />**, чтобы открыть [<Constant name="explorer" />](/docs/explore/explore-projects).
