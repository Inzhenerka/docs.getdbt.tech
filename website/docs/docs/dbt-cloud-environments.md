Окружение определяет, как <Constant name="cloud" /> будет выполнять ваш проект в [<Constant name="cloud_ide" />](/docs/cloud/studio-ide/develop-in-studio) или через [<Constant name="cloud" /> CLI](/docs/cloud/cloud-cli-installation) (для разработки), а также в запланированных заданиях (для деплоя).

Критически важно, что для выполнения dbt среды определяют три переменные:

1. Версия <Constant name="core" />, которая будет использоваться для запуска вашего проекта
2. Параметры подключения к хранилищу данных (включая настройки целевой базы данных и схемы)
3. Версия вашего кода, которая будет выполняться

Каждый проект <Constant name="cloud" /> может иметь только одну [среду разработки](#create-a-development-environment), но количество [сред развертывания](/docs/deploy/deploy-environments) не ограничено. Это дает вам гибкость и возможность настраивать выполнение запланированных заданий под ваши потребности.

Используйте среды для настройки параметров на разных этапах вашего проекта и оптимизации процесса выполнения, применяя принципы инженерии программного обеспечения.

## Создание среды разработки

Чтобы создать новую среду разработки <Constant name="cloud" />:

### Установка учетных данных разработчика

Чтобы использовать dbt <Constant name="cloud_ide" /> или <Constant name="cloud_cli" />, каждому разработчику необходимо настроить [персональные учетные данные для разработки](/docs/cloud/studio-ide/develop-in-studio#get-started-with-the-cloud-ide) для подключения к своему хранилищу данных в разделе **Profile Settings**. Это позволяет задать отдельную целевую конфигурацию (target) и поддерживать индивидуальные учетные данные для подключения к хранилищу.

<Lightbox src="/img/docs/dbt-cloud/refresh-ide/new-development-environment-fields.png" width="85%" height="200" title="Создание среды разработки"/>

Среды развертывания в dbt Cloud необходимы для выполнения запланированных заданий и использования других функций (например, различных рабочих пространств для разных задач). Вы можете иметь множество сред в одном проекте dbt Cloud, что позволяет настроить каждое пространство в соответствии с различными потребностями (такими как эксперименты или тестирование).

Среды развертывания в <Constant name="cloud" /> необходимы для выполнения запланированных заданий и использования других возможностей (например, отдельных рабочих пространств для разных задач). В одном проекте <Constant name="cloud" /> можно создать несколько сред, что позволяет настраивать каждую из них в соответствии с разными потребностями (например, для экспериментов или тестирования).

Несмотря на то что сред может быть много, только одна из них может быть «основной» (main) средой развертывания. Она считается вашей «production»-средой и представляет собой «источник истины» (source of truth) для проекта — именно здесь находятся самые надежные и окончательные преобразования данных.

Чтобы узнать больше о средах развертывания <Constant name="cloud" /> и их настройке, см. страницу [Deployment environments](/docs/deploy/deploy-environments). С рекомендациями по лучшим практикам можно ознакомиться в руководстве [<Constant name="cloud" /> environment best practices](/guides/set-up-ci).

## История настроек среды

Вы можете просматривать историю изменений настроек среды за последние 90 дней.

Чтобы открыть историю изменений:
1. В главном меню перейдите в **Orchestration** и нажмите **Environments**.
2. Нажмите на **название среды**.
3. Нажмите **Settings**.
4. Нажмите **History**.
