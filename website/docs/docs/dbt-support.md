Поддержка dbt доступна всем пользователям через следующие каналы:

- Выделенная команда поддержки dbt (для пользователей <Constant name="cloud" />).
- [Форум сообщества](https://discourse.getdbt.com/).
- [Slack-сообщество dbt](https://www.getdbt.com/community/join-the-community/).
  
## Поддержка dbt Core {#dbt-core-support}

Если вы разрабатываете в командной строке (CLI) и у вас есть вопросы или нужна помощь &mdash; обратитесь к отзывчивому сообществу dbt через [Форум сообщества](https://discourse.getdbt.com/) или [Slack-сообщество dbt](https://www.getdbt.com/community/join-the-community/).

## Поддержка платформы dbt {#dbt-platform-support}

Глобальная команда поддержки dbt доступна клиентам <Constant name="cloud" /> по [email](mailto:support@getdbt.com) или через кнопку **Create a support ticket** в навигации <Constant name="cloud" />.

### Создание обращения в поддержку {#create-a-support-ticket}
Чтобы создать обращение в поддержку в <Constant name="cloud" />:
1. В навигации <Constant name="cloud" /> нажмите **Help & Guides**.
2. Нажмите **Create a support ticket**.
3. Заполните форму и нажмите **Create Ticket**.
4. Сотрудник команды поддержки dbt ответит на ваш запрос по электронной почте.

<Lightbox src="/img/create-support-ticket.gif" title="Создание обращения в поддержку в dbt" />

### dbt Support Assistant {#ask-dbt-support-assistant}
dbt Support Assistant — это AI-виджет, который предоставляет мгновенные, сгенерированные ИИ ответы на распространённые вопросы. Эта функция доступна пользователям <Constant name="cloud" /> и может помочь с вопросами по устранению неполадок, дать краткое описание возможностей и функциональности или указать на релевантную документацию. 

AI-виджет dbt Support Assistant является отдельным инструментом и не связан с [<Constant name="copilot" />](/docs/cloud/dbt-copilot) — мощным AI-движком, который помогает с генерацией кода и ускоряет аналитические рабочие процессы. dbt Support Assistant сосредоточен на ответах, связанных с документацией и устранением проблем. Включение или отключение AI-функций в <Constant name="cloud" /> не влияет на доступность dbt Support Assistant.

:::info
Мы рекомендуем проверять информацию, полученную из AI-ответов, используя нашу документацию. Пожалуйста, [свяжитесь с поддержкой](mailto:support@getdbt.com), чтобы сообщить о некорректной информации, предоставленной Support Assistant.
:::

### Планы поддержки и ресурсы {#support-plans-and-resources}

Мы хотим помочь вам с внедрением и использованием платформы <Constant name="cloud" /> в вашей организации. Есть вопрос, на который вы не нашли ответ в [нашей документации](https://docs.getdbt.com/) или на [Форуме сообщества](https://discourse.getdbt.com/)? Наша команда поддержки всегда готова `dbt help` вам!

- **Планы Enterprise и Enterprise+** &mdash; приоритетная [поддержка](#severity-level-for-enterprise-support), дополнительные премиальные планы, расширенные SLA, помощь во внедрении, выделенное управление аккаунтом и проверки безопасности dbt Labs в зависимости от уровня тарифа.
- **Планы Developer и Starter** &mdash; поддержка 24x5 (без соглашения об уровне сервиса (SLA); по вопросам Enterprise-плана [свяжитесь с отделом продаж](https://www.getdbt.com/pricing/)).
- **Помощь команды поддержки** &mdash; помощь с [типовыми вопросами по <Constant name="cloud" />](/category/troubleshooting), такими как настройка проекта, проблемы со входом, понимание ошибок, настройка приватных пакетов, привязка нового аккаунта GitHub, [как сгенерировать HAR-файл](/faqs/Troubleshooting/generate-har-file) и т.д.
- **Руководство по ресурсам** &mdash; ознакомьтесь с [гайдом](/community/resources/getting-help), чтобы узнать, как эффективнее обращаться за помощью.

## Enterprise-аккаунты dbt {#dbt-enterprise-accounts}

Базовая помощь в устранении проблем с проектами dbt.  
Помощь с ошибками и проблемами в макросах, моделях и пакетах dbt Labs.  
Для стратегических рекомендаций, лучших практик или обсуждений расширения используйте поддержку вашей команды по работе с аккаунтом.

Для клиентов с тарифом <Constant name="cloud" /> уровня Enterprise мы **также** предоставляем базовую помощь в устранении проблем в вашем dbt-проекте:
- **Что-то работает не так, как я ожидал(а)...**
    - в макросе, который я создал(а)...
    - в инкрементальной модели, которую я разрабатываю...
    - в одном из пакетов dbt Labs, таких как dbt_utils или audit_helper...
- **Мне нужна помощь в понимании и устранении этой ошибки...**
    - `Server error: Compilation Error in rpc request (from remote system)
    'dbt_utils' is undefined`
    - `SQL compilation error: syntax error line 1 at position 38 unexpected '<EOF>'.`
    - `Compilation Error Error reading name_of_folder/name_of_file.yml - Runtime Error Syntax
        error near line 9`

Типы вопросов, которые следует адресовать вашей Account-команде:
- Как лучше выстроить наши dbt-проекты, окружения и задания с учётом структуры и потребностей компании?
- Я хочу расширить наш аккаунт! Как добавить больше пользователей и обучить их?
- Вот наш дата-роадмап на следующий год — можем ли мы обсудить, как dbt вписывается в него и какие возможности мы пока не используем?
- Пришло время продления контракта — какие у нас есть варианты?

### Уровни серьёзности для поддержки Enterprise {#severity-level-for-enterprise-support}

Обращения в поддержку получают уровень серьёзности в зависимости от влияния проблемы на ваш бизнес. Уровень серьёзности назначается dbt Labs и определяет приоритет поддержки, который вы получите. Для информации о времени ответа по тикетам или других вопросах, связанных с SLA вашего аккаунта Enterprise или Enterprise+, пожалуйста, обратитесь к вашему контракту Enterprise.

| Уровень серьёзности | Описание |
| -------------- | ----------- |
| Severity Level 1 | Любая ошибка, делающая использование или продолжение использования подписки или её ключевых функций невозможным; подписка не работает, альтернативы отсутствуют. | 
| Severity Level 2 | Отказ функции без обходного решения, при этом подписка остаётся работоспособной. | 
| Severity Level 3 | Отказ функции, для которого существует обходное решение. | 
| Severity Level 4 | Ошибка с минимальным или нулевым влиянием на доступ клиента к подписке или её использование, либо общий вопрос или запрос на улучшение функциональности. | 

## Обратная связь {#leave-feedback}

Оставьте отзыв или отправьте запрос на новую функцию для <Constant name="cloud" /> или <Constant name="core" />.

#### Отправить отзыв или запрос на функцию для платформы dbt {#share-feedback-or-feature-request-for-the-dbt-platform}
1. В навигации <Constant name="cloud" /> нажмите **Leave feedback**.
2. В всплывающем окне **Leave feedback** заполните форму.
3. Прикрепите любые релевантные файлы к форме обратной связи (необязательно).
4. Укажите, хотите ли вы, чтобы dbt Labs связались с вами по поводу отзыва (необязательно).
5. Нажмите **Send Feedback**.

<Lightbox src="/img/docs/leave-feedback.gif" title="Отправка обратной связи в dbt" />

#### Отправить отзыв или запрос на функцию для dbt Core {#share-feedback-or-feature-request-for-dbt-core}
- [Создайте issue на GitHub здесь](https://github.com/dbt-labs/dbt-core/issues).

## Внешняя помощь {#external-help}

Для написания SQL, обзора производительности проекта или построения проекта обратитесь к рекомендованным консультантам dbt и сервисам dbt Labs.  
Если вам нужна помощь с написанием SQL, оценкой общей производительности проекта или вы хотите, чтобы кто-то помог вам непосредственно с построением dbt-проекта, ознакомьтесь со следующими страницами:
- Список [консалтинговых партнёров dbt](https://www.getdbt.com/partner-directory).
- [Сервисы] dbt Labs (https://www.getdbt.com/dbt-labs/services/).
