Объект <Term id="data-lineage">lineage</Term> позволяет выполнять запросы lineage (происхождения и зависимостей данных) для ваших ресурсов. 

В разделе [Пример запроса](#example-query) показаны некоторые поля, которые можно запрашивать с помощью объекта `lineage`. Полную схему со всеми доступными для запроса полями см. в разделе [Поля](#fields).

### Аргументы

При выполнении запроса к `lineage` можно использовать следующие аргументы:

<NodeArgsTable parent="AppliedState" name="lineage" />

### Пример запроса

Вы можете указать `environmentId` и отфильтровать по типу ресурса `"Model"`, чтобы получить информацию о lineage для всех моделей в этом окружении, включая их зависимости, тип материализации и метаданные:

```graphql
query {
  environment(id: 834) {
    applied {
      lineage(
        filter: {"types": ["Model"]} # Return results for the Model type
      ) {
        name
        resourceType
        filePath
        projectId
        materializationType
        parentIds
        tags
        uniqueId
      }
    }
  }
}
```
