Объект snapshots позволяет запрашивать информацию обо всех снапшотах в рамках заданного job.

### Аргументы {#arguments}

При выполнении запроса к `snapshots` доступны следующие аргументы.

<NodeArgsTable parent="JobNode" name="snapshots" />

Ниже приведены несколько наглядных примеров запросов, а также описана схема объекта snapshots.

### Пример запроса {#example-query}

Аргументы database, schema и identifier являются необязательными. Это означает, что с помощью этого эндпоинта вы можете:

- Найти конкретный снапшот, указав `<database>.<schema>.<identifier>`
- Найти все снапшоты в базе данных и/или схеме, указав `<database>` и/или `<schema>`

#### Получение информации о снапшотах для job {#find-snapshots-information-for-a-job}

Пример запроса ниже возвращает информацию обо всех снапшотах в рамках данного job.

```graphql
{
  job(id: 123) {
    snapshots {
      uniqueId
      name
      executionTime
      environmentId
      executeStartedAt
      executeCompletedAt
    }
  }
}
```
