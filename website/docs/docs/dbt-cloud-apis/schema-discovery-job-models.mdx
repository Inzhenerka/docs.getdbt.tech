Объект models позволяет запрашивать информацию обо всех моделях в рамках конкретного job.

### Аргументы

При выполнении запроса к `models` доступны следующие аргументы.

<NodeArgsTable parent="JobNode" name="models" />

Ниже приведены примеры запросов и описание схемы объекта models.

### Примеры запросов
Аргументы database, schema и identifier являются необязательными. Это означает, что с помощью данного эндпоинта вы можете:

- Найти конкретную модель, указав `<database>.<schema>.<identifier>`
- Найти все модели в базе данных и/или схеме, указав `<database>` и/или `<schema>`

#### Поиск моделей по базе данных, схеме и идентификатору
В примере ниже показан запрос, который находит модель по её уникальной базе данных, схеме и идентификатору.

```graphql
{
  job(id: 123) {
    models(database:"analytics", schema: "analytics", identifier:"dim_customers") {
      uniqueId
    }
  }
}
```

#### Поиск моделей по схеме
В примере ниже показан запрос, который находит все модели в данной схеме и их соответствующее время выполнения.

```graphql
{
  job(id: 123) {
    models(schema: "analytics") {
      uniqueId
      executionTime
    }
  }
}
```

### Поля
Объект models предоставляет доступ к _тем же полям_, что и [узел Model](/docs/dbt-cloud-apis/discovery-schema-job-model). Разница заключается в том, что объект models возвращает список, поэтому вместо запроса полей для одной конкретной модели вы можете запросить эти параметры для всех моделей в рамках jobID, базы данных и так далее.
