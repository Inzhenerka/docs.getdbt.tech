# Semantic Layer APIs <Lifecycle status="self_service,managed,managed_plus" />
 
Быстрый рост количества инструментов в современном data-стеке помог специалистам по данным удовлетворять разнообразные потребности разных команд. Обратной стороной этого роста стала фрагментация бизнес-логики между командами, инструментами и рабочими нагрузками.<br /><br />

[<Constant name="semantic_layer" />](/docs/use-dbt-semantic-layer/dbt-sl) позволяет определять метрики в коде (с помощью [MetricFlow](/docs/build/about-metricflow)) и динамически генерировать и запрашивать наборы данных в downstream-инструментах на основе управляемых dbt-активов, таких как метрики и модели. Интеграция с <Constant name="semantic_layer" /> помогает организациям, использующим ваш продукт, принимать более эффективные и заслуживающие доверия решения на основе данных. Кроме того, это позволяет избежать дублирования кода, оптимизировать процесс разработки, обеспечить управление данными и гарантировать согласованность данных для их потребителей.  

Вы можете использовать <Constant name="semantic_layer" /> для самых разных инструментов и сценариев работы с данными. Вот некоторые распространённые варианты использования:

Вы можете использовать семантический слой dbt для различных инструментов и приложений данных. Некоторые распространенные случаи использования:

* Бизнес-аналитика (BI), отчетность и аналитика
* Качество данных и мониторинг
* Управление и конфиденциальность
* Обнаружение данных и каталогизация
* Машинное обучение и наука о данных
